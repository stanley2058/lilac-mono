# HEARTBEAT.md

This file is the durable state for the autonomous heartbeat lane.

## Ownership

- If you are not running in heartbeat mode, do not edit this file directly.
- Non-heartbeat runs should hand work to `heartbeat/inbox/` with one file per note.
- Heartbeat mode owns this file and is responsible for merging inbox notes and archiving processed notes.

## Inbox Note Template

Write one markdown file per handoff note in `heartbeat/inbox/`.

```md
---
createdAt: 2026-03-11T10:15:00.000Z
sourceSessionId: "<session-id>"
sourceRequestId: "<request-id>"
kind: "watch"
priority: "normal"
---
What to monitor, when to follow up, and why it matters.
```

## Heartbeat Rules

- Start by reading this file and checking `heartbeat/inbox/`.
- Merge durable work into the sections below.
- After processing an inbox note, move it into `heartbeat/archive/`.
- Keep this file concise and current.
- Use files as memory; do not rely on prior transcript history.
- Prefer the default proactive output session from the trigger prompt when you need to send a heartbeat message.
- If you need to tell a human something, use the surface tool. Normal assistant output is discarded, so finish with `HEARTBEAT_OK`.
- During soft quiet hours, avoid low-priority proactive outreach unless the issue is urgent, critical, or time-sensitive.
